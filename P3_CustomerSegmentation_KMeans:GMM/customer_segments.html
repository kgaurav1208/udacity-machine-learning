Implementation: Outlier Detection¶

Detecting outliers in the data is extremely important in the data preprocessing step of any analysis. The presence of outliers can often skew results which take into consideration these data points. There are many "rules of thumb" for what constitutes an outlier in a dataset. Here, we will use Tukey's Method for identfying outliers: An outlier step is calculated as 1.5 times the interquartile range (IQR). A data point with a feature that is beyond an outlier step outside of the IQR for that feature is considered abnormal.

In the code block below, you will need to implement the following:

  • Assign the value of the 25th percentile for the given feature to Q1. Use np.percentile for this.
  • Assign the value of the 75th percentile for the given feature to Q3. Again, use np.percentile.
  • Assign the calculation of an outlier step for the given feature to step.
  • Optionally remove data points from the dataset by adding indices to the outliers list.

NOTE: If you choose to remove any outliers, ensure that the sample data does not contain any of these points!
Once you have performed this implementation, the dataset will be stored in the variable good_data.

Answer: K-means clustering algorithms have significantly lesser parameters (cluster centers and cluster assignments) to learn compared to Gaussian Mixture Model, which must also learn covariance matrices for clusters in addition to the cluster assignments and centers. This makes the K-means algorithm much faster and less resource hungry, whenever the clusters are isotropic, have no or little overlap and all prior probabilities are even. Gaussian Mixture Model, despite being slow and resource hungry, is a richer model and is more appropriate whenever the underlying clusters do not satisfy the aforementioned conditions. Another advantage of Gaussian Mixture Model is that it provides 'soft cluster assignments' instead of 'hard assignments' as in the case of K-Means, that is, it gives the probability for a point to belong to any cluster. This gives us the confidence value for every cluster assignment and could be useful when underlying clusters have significant overlap and/or dataset is small.

Our first preference would be to run the K-Means algorithm, simply because it is really fast. If it would seem that the results could be significantly improved by using Gaussian Mixture Model, then it could be used in the next step, using the cluster assignments and centers obtained from K-means as the initialization for Gaussian Mixture Model. Observing the sample points, sample point 0 having relatively high value of 'Detergents_Paper' and low value of 'Fresh'(and thus large PCA-1 component and small PCA-2 component) seems to be well separated from sample point 2 which has high value of 'Fresh' and low value of 'Detergents_Paper' (and thus large PCA-2 component and small PCA-1 component). Sample point 1 would fall somewhere in the middle on both axis, but a casual glance at the dataset indicates that the proportion of such points in the data is not huge. Therefore, the initial impression is that the K-Means algorithm could give satisfactory results.

Question 10¶

Companies often run A/B tests when making small changes to their products or services. If the wholesale distributor wanted to change its delivery service from 5 days a week to 3 days a week, how would you use the structure of the data to help them decide on a group of customers to test?
Hint: Would such a change in the delivery service affect all customers equally? How could the distributor identify who it affects the most?

Answer: In A/B testing, we test two versions of a hypothesis, say A and B, on two sets of randomly chosen samples from the target population. In the context of the question, A would be delivery service of 3 days a week and B would be delivery service of 5 days a week. The trade-off among the two choices would be reduced delivery costs in the case of A, while daily 'Fresh' supplies in the case of B. It is reasonable to assume that the retailers would prefer the option A, while the restaurants have a strong preference for option B. If the two test sets are chosen from the entire set of customers while ignoring the structure in the data, it is possible that the preferences of two kinds of customers cancel each other, therefore the difference between A and B might not come out to be statistically significant. Therefore, it is better to run A/B testing separately for the two clusters of the customers. By taking two random sets from each cluster and testing A and B on each pair of sets, one might discover statistically siginificant preference for one over another in both or one of the clusters. In this specific case, one would expect that A/B testing on the cluster corresponding to restaurants will reveal statistically significant preference for B (daily 'Fresh' supplies).
